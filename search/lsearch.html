Linear Search

☰ Home | Searching Algorithms | Sorting Algorithms | Graph Algorithms | Greedy Algorithms

Description:

Linear Search is the simplest and most basic searching algorithm. It traverses the dataset sequentially, examining each element one by one until the desired value is found or the entire dataset is checked. This makes it straightforward but less efficient for larger datasets.

Time Complexity:

  • Best Case: O[1] (when the target element is the first item)
  • Worst Case: O[n] (when the target element is at the end or not found)
  • Average Case: O[n]

Space Complexity:

  • O[1] (in-place, does not require additional space)

Used for:

  • Small or unsorted datasets where simplicity outweighs the need for efficiency.

Applications:

  • Finding Elements in Unsorted Data: For cases where data isn't sorted, Linear Search is often the simplest choice.
  • Small Data Sizes: It performs well for small datasets where setup complexity for more advanced algorithms isn't justified.
  • Basic Educational Purposes: Its simplicity makes it ideal for teaching the basics of searching algorithms in computer science.
