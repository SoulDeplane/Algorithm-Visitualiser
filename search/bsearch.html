Binary Search

☰ Home | Searching Algorithms | Sorting Algorithms | Graph Algorithms | Greedy Algorithms

Description:

Binary Search is a highly efficient algorithm designed for sorted datasets. It works by repeatedly dividing the search space in half, comparing the target value to the middle element of the current range, and deciding whether to search the left or right half. This halving approach significantly reduces the number of comparisons.

Time Complexity:

  • Best Case: O[1] (when the middle element matches the target)
  • Worst Case:O[log n] (when the target element is found after searching all levels of the binary tree)
  • Average Case: O[log n]

Space Complexity:

  • O[1] (iterative version)
  • O[log n] (recursive version, due to the call stack)

Used for:

  • Efficient searching in large, sorted datasets.

Applications:

  • Searching in Large Datasets: Ideal for datasets where sorting is already done, such as numerical arrays or dictionaries.
  • Efficient Lookup: Often used in applications like searching for words in a dictionary, database indexing, or locating elements in a sorted list.
  • Divide-and-Conquer Problems: A foundational technique for problems involving recursion and halving strategies.
